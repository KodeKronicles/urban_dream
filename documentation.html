Project Overview

The homepage follows a minimalistic yet engaging design to balance aesthetics and usability. A flowing grid animation showcases our chosen objects in a subtle yet catchy way. The lowered opacity ensures that while the images catch the eye, they do not distract from the overall experience.

Below the main section, a brief ‘About’ section provides context for the project’s purpose, followed by an engaging timeline that highlights key urban planning milestones. At the very bottom, a map along with the museum's opening hours guides the user, and the footer features a clickable GitHub logo linking to the project repository.

Content Structure

Site architecture

The architecture of the website is designed to guide users through a virtual exhibition, with each page being designed for an intuitive experience.

Navigation bar

The anchoring point between all pages is a simple navigation bar, with a version that is slightly different visually being prepared for the home page for more visual distinction.

On the left side of the navbar, we have placed the logo of the project, which also doubles as a home page button. On its' right side, there are five links, to Home, Exhibition, Map, Items and About pages respectively.

The five-item list on the right transforms into a hamburger-button list on smaller widths to accommodate mobile devices.

Home page

The home page serves as an introduction to the virtual exhibitions, with the images of all the items presented via a three-row horizontal parallax scrolling div. Below it, a vertical timeline introducing the chief advancements in urbanism is presented. The home page culminates with a map showing the location of the exhibition and the opening pages.

Exhibition

The exhibition page serves as further introduction to the topic of the exhibition. There are five groups of items, further divided into subgroups (eg. location -> Europe). Clicking an item here redirects to a page containing its longest description.

Map

The map page is divided into two parts; the map proper, and the list of all items on exhibition. The map contains 20 clickable elements, each corresponding to an exhibition item, which filter the underlying list to show just one item.

The list contains all 20 exhibition items with 6 attributes, on which the list can be sorted. Clicking an item extends a tab with additional information, both visual and textual, and an external link for further reference.

Item viewer

The item viewer page presents images and metadata of all exhibition items. There is one image per item, and its' display on the page is standardised. The item metadata contains information about time, place, type and clickable links to narratives. In the same section, below the metadata table, there are two button groups; the first one for changing the displayed item in the current narrative to next/previous, and the second for indicating and toggling between items in the current narrative.

The section below contains textual information about the items, and the buttons to display more/less information.

Footer

The page contains a simple footer, showing the copyright and a link to the GitHub repository containing the project.

Design Elements

Layout

The homepage layout embraces a minimalistic design with a clear timeline guiding users through historical urban planning milestones. A grey vertical block is strategically positioned slightly to the left of the flowing grid, ensuring that the project name remains prominent while the imagery is allowed to breathe.

Grid Layout and Spacing

Bootstrap Containers:

  • The container-fluid class ensures full-width adaptability, optimizing space usage.
  • A combination of col-md-2 for side elements and col-md-10 for main content balances proportions and maintains an intuitive navigation flow.
  • Bootstrap's grid system facilitates responsiveness by adjusting layouts based on screen width.

Margin and Padding Consistency:

  • Fixed units (px, rem) are used for elements requiring precise control, such as border widths and icons.
  • Relative units (%, em, vh, vw) enhance flexibility, ensuring smooth layout adaptation across devices.
  • Margins and paddings are carefully structured to avoid visual clutter while maintaining alignment and readability.

Recurring Percentages:

  • 7% for larger section separations, centering content and ensuring visual hierarchy.
  • 5% for moderate spacing within content containers, maintaining readability and structure.
  • 3% for tighter spacing, particularly between text and images, optimizing information density without overwhelming the user.

Typography and Colors

Fonts:

  • Inter – used for the navbar and general text, ensuring readability and a modern look.
  • Futura Maxi CG Bold – used for project titles, providing a bold and striking appearance.

Color Palette:

  • Primary Color: Deep red (#D02E02) is used for accents and branding.
  • Secondary Colors: Grey (#E9E9E9) and Black maintain contrast and readability.
  • User Feedback: Prolonged exposure to red text can cause eye strain; therefore, large text blocks may be transitioned to a more neutral red or grey shade.

Functionality

Interactive Features

Flowing Grid Animation:

  • Implemented using CSS Grid Layout to create a structured and responsive design.
  • CSS transitions and opacity settings ensure a smooth scrolling effect with a subtle fade.
  • A JavaScript animation loop dynamically updates the grid’s position, creating an infinite horizontal scroll.
  • Images maintain their aspect ratio and responsiveness using object-fit: cover.

Parallax Scrolling:

  • Implemented using CSS and JavaScript to create a dynamic depth effect as users scroll.
  • Background images move at a different speed than foreground content, enhancing the immersive experience.
  • Smooth scrolling transitions ensure the effect is visually appealing without disrupting readability.
  • This effect is applied to key sections, such as the About and Map areas.

Navbar and Responsiveness:

  • The navbar dynamically adapts to different screen sizes.
  • A custom navbar toggler improves the mobile experience.
  • Dropdown width calculations prevent layout shifts when resizing.

Timeline Section:

  • Implemented using CSS animations and ScrollMagic.js.
  • Timeline entries fade in when scrolled into view, creating a dynamic storytelling effect.
  • Alternating even and odd items enhance readability and structure.

Counter Animation:

  • Dynamically counts to 19 to highlight exhibit items.
  • Triggers when the element enters the viewport.
  • Uses aria-hidden attributes to ensure accessibility.

Mobile Optimization:

  • Grid images resize based on screen width.
  • On smaller screens, the timeline collapses into a single column and the red timeline line is hidden.
  • The navbar toggler is optimized for smooth mobile interaction.

Responsiveness

The layout of the website has been developed to accommodate different screen sizes, in order to ensure a consistent browsing experience for users on computers and mobile devices. This has been achieved using:

  • CSS media queries: used to customise styling based on device characteristics (in our case: resolution).
  • Flex grid system: Bootstrap's flex grid systems allows for easy adaptation of content display according to used resolution.
  • Scalable components Some components used adjust dynamically to the viewport size, with margins and paddings scaling appropriately.
  • Responsive images: Images are resized according to the resolution used.

Workflow

User Survey & Improvements

  • Scaling issues and bugs on the homepage: Currently being addressed.
  • Footer style inconsistency: Adjustments have been made for uniform styling.
  • Readability concerns: Users reported that prolonged exposure to red text can cause eye strain. Large text blocks may be transitioned to a more neutral red or grey shade.
  • Table in “Map” section:
    • 3 out of 4 users suggested it should be open by default.
    • 1 user recommended improving the button prompt for better visibility.
  • Issues with tapping the “+” button on mobile: Further testing is required for an optimal solution.
  • Mobile map margins: Adjustments will be made to add more breathing space on the sides.
  • Anchor texts: Hovering over hyperlinks does not provide anchor links, which could be used to describe the redirection.
